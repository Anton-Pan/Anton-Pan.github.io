<a href="https://www.youtube.com/watch?v=b06pKMxF6h8">Vir</a><a href="https://www.youtube.com/watch?v=HPc8QMycGno">tua</a><a href="https://www.youtube.com/watch?v=e7QeR0VxnlA">l S</a><a href="https://www.youtube.com/watch?v=zdLhJ9lB5x0">elf</a>
musical project.
<br/>
<br/>
I've hand-written <b>all</b> the code for this site.
<br/>
No libraries or frameworks. Just pure HTML, CSS and JavaScript.
<br/>
I have two reasons for this choice:
<br/>
The first is <b>learning</b>: I consider this site to be, in addition to a portfolio, a learning experience.
Having spent so much time maximizing efficiency through libraries and frameworks, I decided to go back to the basics and learn from the ground up
to strengthen my foundational knowledge.
<br/>
The second is <b>limit-testing</b>: I want to see just how far I can push my skills and creativity without using any frameworks, libraries, or copied code.
<br/>
For example; the falling snow is just multiple CSS animations that transition between two random points at the top and bottom of the screen,
which are randomly regenerated with every individual animation completion.
